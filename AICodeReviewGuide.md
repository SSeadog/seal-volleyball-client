Unity AI Code Review Guidelines

이 문서는 AI가 Unity 프로젝트의 C# 코드를 일관되고 효과적으로 리뷰할 수 있도록 하기 위한 기준입니다.

⸻

1. 아키텍처 / 구조

체크 항목
	•	클래스가 단일 책임 원칙(SRP)을 지키고 있는가?
	•	MonoBehaviour가 과도하게 많은 역할을 담당하고 있지 않은가?
	•	비즈니스 로직이 View(MonoBehaviour)에 과하게 포함되어 있지 않은가?
	•	클래스 간 의존성 방향이 올바른가?
	•	순환 참조가 존재하지 않는가?
	•	인터페이스 또는 DI(의존성 주입)를 활용할 수 있는 구조인가?

리뷰 포인트
	•	역할이 명확하지 않은 클래스 분리 필요 여부
	•	테스트 가능한 구조인지 여부

⸻

2. Unity 생명주기 사용

체크 항목
	•	Update / FixedUpdate / LateUpdate 사용이 적절한가?
	•	불필요한 Update 사용이 존재하는가?
	•	Awake와 Start의 역할이 올바르게 구분되어 있는가?
	•	OnEnable / OnDisable에서 이벤트 구독/해제가 제대로 이루어지는가?

리뷰 포인트
	•	매 프레임 실행될 필요 없는 로직이 Update에 포함되어 있는지 확인
	•	이벤트 해제 누락으로 인한 메모리 문제 여부

⸻

3. 성능

체크 항목
	•	GC Allocation을 유발하는 코드가 존재하는가?
	•	LINQ 사용
	•	반복적인 new 객체 생성
	•	GetComponent를 반복 호출하고 있는가?
	•	string 연산이 과도하게 사용되고 있는가?
	•	Instantiate / Destroy 호출이 잦은가?
	•	Find / FindObjectOfType 사용 여부

리뷰 포인트
	•	캐싱이 필요한 부분 식별
	•	Object Pooling 적용 가능 여부
	•	프레임 드랍 가능성 있는 코드 식별

⸻

4. 비동기 / 코루틴

체크 항목
	•	Coroutine이 불필요하게 많이 사용되고 있는가?
	•	StopCoroutine이 누락된 부분이 있는가?
	•	async/await와 Coroutine이 혼용되어 문제를 유발할 가능성이 있는가?
	•	race condition 발생 가능성이 있는가?

리뷰 포인트
	•	흐름 제어가 명확한지
	•	리소스 정리가 제대로 되는지

⸻

5. 코드 스타일 / 가독성

체크 항목
	•	변수명과 함수명이 명확하고 의미 있는가?
	•	매직 넘버가 존재하는가?
	•	함수가 지나치게 길지 않은가?
	•	중복 코드가 존재하는가?
	•	early return이 적절히 사용되었는가?

리뷰 포인트
	•	가독성 향상 가능 여부
	•	유지보수성 개선 가능 여부

⸻

6. 예외 처리 / 안정성

체크 항목
	•	null 체크가 충분히 이루어지고 있는가?
	•	try-catch가 필요한 곳에 존재하는가?
	•	Inspector에서 할당되는 값에 대한 안전성 확보 여부

리뷰 포인트
	•	런타임 에러 가능성
	•	방어 코드 부족 여부

⸻

7. 테스트 가능성

체크 항목
	•	단위 테스트가 가능한 구조인가?
	•	MonoBehaviour에 로직이 과도하게 의존하고 있는가?
	•	순수 클래스(pure class)로 분리 가능한 부분이 있는가?

리뷰 포인트
	•	테스트 용이성 개선 가능 여부

⸻

8. Unity 에디터 / 직렬화

체크 항목
	•	SerializeField를 적절히 사용하고 있는가?
	•	public 필드가 불필요하게 사용되고 있지 않은가?
	•	ScriptableObject 활용 가능성이 있는가?

리뷰 포인트
	•	Inspector 친화적인 구조인지
	•	데이터 관리 방식 개선 가능 여부

⸻

9. 메모리 / 리소스 관리

체크 항목
	•	이벤트 해제가 누락된 부분이 있는가?
	•	리소스 참조가 해제되지 않는 문제가 있는가?
	•	Addressables 또는 Resources 사용이 적절한가?

리뷰 포인트
	•	메모리 누수 가능성
	•	리소스 관리 전략 개선 여부

⸻

권장 출력 형식 (AI용)

AI는 아래 형식으로 리뷰를 수행한다:
	•	문제점
	•	문제 원인
	•	개선 방법 (코드 예시 포함)
	•	심각도 (Low / Medium / High)

⸻

핵심 원칙 요약
	1.	Unity 생명주기를 잘못 사용하는 부분을 우선적으로 찾는다.
	2.	성능 문제 (특히 GC, Update, 반복 호출)를 집중적으로 분석한다.
	3.	유지보수성과 구조 개선 가능성을 항상 고려한다.
